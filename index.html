How it works

Three steps from infrastructure code to a shareable architecture diagram.

STEP 01

Paste your IaC code

Copy your Terraform HCL or docker-compose.yml into the editor. No reformatting needed — InfraSketch parses it as-is, including partial files and modules.

›
STEP 02

Auto-detect architecture

The parser reads VPC boundaries, subnet membership (subnet_id, subnet_ids), and resource references to build a containment graph — separating structural wiring from actual traffic flow.

›
STEP 03

Export and share

Download as PNG for presentations, SVG for docs and READMEs, or draw.io XML to keep editing in diagrams.net. All exports are client-side — nothing is uploaded.

Everything you need

Built for engineers who want diagrams without the overhead of visual tools.

No credentials needed

Unlike Cloudcraft or Hava, InfraSketch never touches your cloud account. Just paste code — everything runs in your browser.

Official AWS icons

Generates diagrams using proper AWS architecture icons. Professional enough for documentation and presentations.

Three export formats

Download as PNG or SVG for docs and slides, or export draw.io XML to keep editing the diagram in diagrams.net.

100% private

Your code never leaves your browser. No server processing, no storage, no telemetry. Fully client-side.

VPC containment layout

Understands that vpc_id and subnet_id are structural — renders subnets as nested boxes inside the VPC, not as arrows.

Clean connection routing

Skips structural wiring references and only draws meaningful flow connections — no spaghetti arrows from every Terraform resource reference.
