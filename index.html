How it works

Three steps from infrastructure code to a shareable architecture diagram.

STEP 01

Paste your IaC code

Paste Terraform HCL, the output of terraform show -json tfplan, or a docker-compose.yml. No reformatting needed — InfraSketch auto-detects the format.

›
STEP 02

Auto-detect architecture

The parser resolves VPC / VNet boundaries, subnet membership, and resource references for AWS and Azure — separating structural wiring from actual traffic flow connections.

›
STEP 03

Export and share

Download as PNG, SVG, or draw.io XML. Hit Share to copy a URL that encodes the full diagram state — anyone with the link sees the same diagram instantly.

Everything you need

Built for engineers who want AWS and Azure diagrams without visual tools or cloud credentials.

⚡

No credentials needed

Unlike Cloudcraft or Hava, InfraSketch never touches your cloud account. Just paste code — everything runs in your browser.

🎨

AWS + Azure icons

Official AWS architecture icons and Microsoft Azure service icons for 50+ resource types across both providers.

📦

Three export formats

Download as PNG or SVG for docs and slides, or export draw.io XML to keep editing the diagram in diagrams.net.

🔗

Shareable URLs

Click Share to encode the diagram state into the URL hash. Send the link — recipients see your exact diagram without any server involved.

📋

Terraform plan JSON

Run terraform show -json tfplan and paste the output directly. Connection accuracy is higher than HCL parsing because plan JSON contains resolved resource references.

💡

VPC / VNet containment

Renders subnets nested inside VPCs and VNets. Supports both vpc_id (AWS) and virtual_network_name (Azure).
